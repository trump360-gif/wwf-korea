## 1. 프로젝트 개요

### 1.1 배경

WWF-Korea(세계자연기금 한국본부)의 온라인 기부 플랫폼을 신규 구축한다. 기부자가 자신의 후원금이 어떤 보전 활동에 쓰이길 원하는지 직접 선택할 수 있는 구조를 중심으로, WWF의 6대 보전 미션을 각각 깊이 있게 전달하는 스토리텔링 랜딩페이지와 기부 플로우, 감사 페이지까지를 하나의 경험으로 설계한다.

### 1.2 핵심 목표

- 기부자에게 **참여감과 연결감**을 제공한다 — 내 돈이 어디에 쓰이는지 선택하는 경험
- **상품 구매가 아닌 마음의 선택**으로 느껴지도록 설계한다
- 각 보전 미션의 이야기를 에디토리얼 수준으로 전달하여 **공감과 이해를 유도**한다
- 어떤 선택을 하든 **감사와 긍정의 메시지**가 전달되어야 한다

### 1.3 결과물 범위

| 항목 | 설명 |
|------|------|
| 메인 랜딩페이지 | WWF 소개 + 6개 미션 카드 진입점 |
| 미션별 랜딩페이지 (×6) | 각 미션의 스크롤 내러티브 + 하단 기부 버튼 |
| 기부 스텝 모달 | 3단계 기부 플로우 (확인 → 추가 선택 → 결제) |
| 감사 페이지 | 기부 완료 확인 + 기증서 PDF 다운로드 |

### 1.4 결과물 형태

실제 동작하는 React 프로토타입 (SPA)

## 2. 사이트 구조

```
메인 랜딩
├── 히어로 섹션 (WWF 소개, 핵심 메시지)
├── 미션 카드 섹션 (6개 카드)
│   ├── 기후·에너지
│   ├── 해양
│   ├── 야생동물
│   ├── 식량
│   ├── 담수
│   └── 산림
└── 푸터 (WWF 정보, 링크)

미션별 랜딩페이지 (×6)
├── 히어로 (전면 사진 + 미션 타이틀)
├── 스크롤 내러티브 섹션들
│   ├── 문제 제기 ("지금 일어나고 있는 일")
│   ├── WWF의 활동 ("우리가 하고 있는 일")
│   └── 참여 호소 ("당신의 마음이 닿을 수 있는 곳")
├── 기부하기 CTA 버튼
└── 푸터

기부 스텝 모달
├── 스텝 1: 미션 확인 + 금액 설정
├── 스텝 2: 추가 미션 참여 여부 (예/아니오)
├── 스텝 2-1: (예 선택 시) 추가 카테고리 선택
└── 스텝 3: 결제 정보 입력

감사 페이지
├── 감사 메시지
├── 기부 요약 (금액, 선택 미션, 날짜)
├── 기증서 PDF 다운로드 버튼
└── 메인으로 돌아가기
```

## 3. 미션 카테고리

WWF 글로벌 공식 6대 보전 프로그램을 그대로 사용한다.

### 3.1 기후·에너지

- 핵심 키워드: 1.5도 목표, 탄소 감축, 재생에너지 전환
- 한국 활동: 어스아워 캠페인, 기후행동 서포터즈, 기후행동 라운드테이블
- 스토리텔링 소재: 지구 평균 기온 상승 데이터, 어스아워 참여 현황, 에너지 전환 사례

### 3.2 해양

- 핵심 키워드: 해양 생태계, 플라스틱 오염, 불법어업
- 한국 활동: 해양 침적 어구 수거, 바다거북 서식지 보호, 해안 정화
- 스토리텔링 소재: 참고래 부검 사례, 해양 쓰레기 1,600kg 수거, 통영 해역 사업

### 3.3 야생동물

- 핵심 키워드: 멸종위기종 보전, 서식지 보호, 생물다양성
- 한국 활동: 애니스테이 캠페인 (까막딱따구리, 꿀벌, 바다거북, 반달가슴곰, 수달), DMZ 두루미 서식지 보전
- 스토리텔링 소재: 광릉숲 유네스코 생물권 보전지역, 철원 DMZ 두루미, 국내 멸종위기종 282종

### 3.4 식량

- 핵심 키워드: 지속가능한 식량 시스템, 농업 전환, 식량 안보
- 한국 활동: 지속가능한 생산 방식 전환, 비즈니스 모델 구축 연구
- 스토리텔링 소재: 글로벌 식량 시스템이 자연에 미치는 영향, 지속가능한 농업 사례

### 3.5 담수

- 핵심 키워드: 하천·습지 보전, 수자원 보호, 담수 생태계
- 한국 활동: 수달 서식지 하천 정화, 인공섬 쉼터 설치
- 스토리텔링 소재: 담수 생태계 위기 데이터, 하천 정화 프로젝트, 수달 모니터링

### 3.6 산림

- 핵심 키워드: 산림 보전, 생태계 복원, 탄소 흡수
- 한국 활동: 광릉숲 생물다양성 증진, 희귀·특산식물 보호
- 스토리텔링 소재: 산림 파괴 데이터, 광릉숲 복원 이야기, 산림의 탄소 흡수 역할

### 3.7 미션 데이터 타입 정의

`lib/data/missions.ts`에서 사용할 타입 명세:

```typescript
interface Mission {
  slug: MissionSlug
  name: string                    // "해양"
  fullName: string                // "해양 보전"
  subtitle: string                // 히어로 서브타이틀: "바다가 보내는 조용한 신호"
  description: string             // 카드용 한 줄 설명 (호버 시 노출)
  icon: string                    // Lucide 아이콘명 (예: "waves")
  heroImage: string               // 히어로 이미지 경로
  cardImage: string               // 메인 랜딩 카드 이미지 경로
  keywords: string[]              // 핵심 키워드 배열
  narrative: {
    problem: NarrativeSection     // "지금 일어나고 있는 일"
    action: NarrativeSection      // "우리가 하고 있는 일"
    appeal: NarrativeSection      // "당신의 마음이 닿을 수 있는 곳"
  }
  stats: Stat[]                   // 통계 하이라이트 (1~3개)
  impactMessage: string           // 기증서용 임팩트 문구
  ctaMessage: string              // CTA 부근 마무리 텍스트
}

interface NarrativeSection {
  heading: string                 // 섹션 제목
  body: string                    // 본문 (마크다운 허용)
  images: NarrativeImage[]        // 섹션 이미지 (1~2장)
  activities?: string[]           // WWF 활동 목록 (action 섹션용)
}

interface NarrativeImage {
  src: string
  alt: string
  caption?: string                // 사진 하단 캡션
  layout: 'full' | 'half' | 'inset'  // full-bleed / 반폭 / 본문 삽입
}

interface Stat {
  value: string                   // "1,600" (카운트업 대상)
  unit: string                    // "kg"
  label: string                   // "수거된 해양 쓰레기"
}

type MissionSlug =
  | 'climate-energy'
  | 'ocean'
  | 'wildlife'
  | 'food'
  | 'freshwater'
  | 'forest'
```

**미션별 임팩트 메시지 (기증서용):**

| slug | impactMessage |
|------|---------------|
| climate-energy | "당신의 후원으로 지구의 온도를 지킵니다" |
| ocean | "당신의 후원으로 해양 생태계가 보호됩니다" |
| wildlife | "당신의 후원으로 야생동물의 서식지가 지켜집니다" |
| food | "당신의 후원으로 지속가능한 식량 시스템을 만듭니다" |
| freshwater | "당신의 후원으로 맑은 물과 생명이 이어집니다" |
| forest | "당신의 후원으로 숲이 다시 숨 쉽니다" |

## 4. 기부 플로우 상세

### 4.1 진입 경로

각 미션 랜딩페이지 하단의 **"기부하기"** 버튼 클릭 시 스텝 모달이 열린다.
진입한 미션이 기본 선택 카테고리로 자동 설정된다.

### 4.2 스텝 1 — 미션 확인 + 금액 설정

**화면 구성:**
- 상단: 진입 미션 확인 메시지 (예: "해양을 위한 후원을 시작합니다")
- 후원 유형 선택: 월정기 후원 / 일시 후원 토글
- 금액 설정:
  - 프리셋 버튼: 1만원 / 3만원 / 5만원 / 10만원
  - 직접 입력 필드
- 하단: "다음" 버튼

**UX 원칙:**
- 금액 프리셋은 WWF-Korea 실제 후원 구간 참고
- 월정기 후원 시 후원품 안내는 이 단계에서 노출하지 않음 (상품화 방지)
- 금액 선택이 "가격표"처럼 보이지 않도록 시각적으로 절제된 디자인

### 4.3 스텝 2 — 추가 미션 참여 여부

**화면 구성:**
- 진입 미션에 대한 감사 메시지 (예: "해양을 향한 마음, 감사합니다")
- 질문: "다른 자연보전 활동에도 함께하시겠어요?"
- 두 개의 선택지:
  - **"네, 더 알아볼게요"** → 스텝 2-1로 이동
  - **"[미션명]에 집중할게요"** → 스텝 3(결제)으로 이동

**UX 원칙:**
- 두 버튼은 동일한 크기와 시각적 비중으로 배치한다 — "아니오"가 열등한 선택으로 느껴지면 안 된다
- "아니오" 선택 시에도 반드시 긍정 메시지를 보여준다 (예: "당신의 마음이 해양에 닿습니다")
- 이 단계는 업셀링이 아니라 **관심의 확장 제안**이다. 카피 톤은 권유가 아닌 안내에 가깝게
- 절대 죄책감을 유발하지 않는다

### 4.4 스텝 2-1 — 추가 카테고리 선택 (조건부)

**화면 구성:**
- 나머지 5개 미션 카테고리가 카드 형태로 노출
- 각 카드: 미션 아이콘 + 미션명 + 한 줄 설명
- 복수 선택 가능 (체크박스)
- 선택 시 기부금은 선택한 전체 미션에 균등 배분
- 하단: "다음" 버튼

**UX 원칙:**
- 카드 선택 시 배분 비율을 시각적으로 보여준다 (예: 3개 선택 → "각 33%씩 배분됩니다")
- 배분 비율은 자동 균등 — 수동 비율 조절은 1.0 범위에서 제외 (복잡도 관리)
- 아무것도 추가 선택하지 않고 "다음"을 누르면 원래 미션 100%로 진행

### 4.5 스텝 3 — 결제 정보 입력

**화면 구성:**
- 기부 요약: 총 금액, 후원 유형(월정기/일시), 배분 미션 목록
- 기부자 정보: 이름, 이메일, 연락처
- 결제 수단 선택 (프로토타입에서는 UI만 구현)
- 하단: "후원하기" 버튼

**참고:** 실제 결제 연동은 프로토타입 범위 밖. UI/UX 흐름만 구현한다.

### 4.6 예외 처리 및 엣지 케이스

#### 금액 입력 검증

| 조건 | 처리 |
|------|------|
| 금액 미입력/0원 | "다음" 버튼 비활성화, 입력 필드 하단 안내: "후원 금액을 선택해주세요" |
| 최소 금액 미달 | 최소 1,000원. 안내: "최소 후원 금액은 1,000원입니다" |
| 최대 금액 초과 | 일시 후원 최대 1억원, 월정기 최대 1,000만원. 안내: "최대 후원 금액을 초과했습니다" |
| 숫자 외 입력 | 입력 자체 차단 (input type=number + 한글/특수문자 필터) |

#### 필수 필드 검증 (스텝 3)

| 필드 | 규칙 | 안내 메시지 |
|------|------|------------|
| 이름 | 2자 이상, 공백만 불가 | "이름을 입력해주세요" |
| 이메일 | 이메일 형식 검증 | "올바른 이메일 주소를 입력해주세요" |
| 연락처 | 숫자 10~11자리, 자동 하이픈 | "연락처를 입력해주세요" |
| 결제 수단 | 하나 이상 선택 필수 | "결제 수단을 선택해주세요" |

- "후원하기" 버튼은 모든 필수 필드가 유효할 때만 활성화
- 인라인 검증 (blur 시 즉시 피드백), 폼 제출 시 재검증

#### 모달 동작 정책

| 상황 | 동작 |
|------|------|
| 모달 닫기 (X 버튼) 클릭 | 스텝 2 이후라면 확인 다이얼로그: "작성 중인 후원 정보가 사라집니다. 닫으시겠어요?" / 스텝 1이면 즉시 닫기 |
| 배경 오버레이 클릭 | 모달 닫기와 동일 |
| 브라우저 뒤로가기 | 모달이 열려있으면 모달만 닫기 (history.pushState로 모달 상태 관리) |
| 모달 닫기 후 재진입 | 상태 초기화 — 처음부터 다시 시작 |
| 페이지 새로고침 | 상태 유실 — 모달 닫힌 상태로 복구 (Zustand persist 미사용) |

#### 감사 페이지 진입 보호

- 기부 완료 상태 없이 `/donate/complete` 직접 접근 시 → 메인 페이지로 리다이렉트
- 감사 페이지 진입 시 Zustand 상태를 `sessionStorage`에 1회성 백업하여, 새로고침 시에도 감사 메시지 표시 가능하도록 처리

## 6. 디자인 가이드라인

### 6.1 디자인 컨셉

**에디토리얼/매거진** 스타일을 기반으로 한다. 내셔널 지오그래픽, 뉴욕타임스 매거진 등의 장편 기사처럼 사진과 타이포그래피가 주도하는 레이아웃을 지향한다. 목적은 정보 전달이 아니라 **경험과 공감**이다.

### 6.2 컬러 시스템

WWF 공식 브랜드 컬러를 기반으로 한다.

| 역할 | 컬러 | 용도 |
|------|------|------|
| Primary Black | `#000000` | 본문 텍스트, 헤드라인, 주요 UI |
| White | `#FFFFFF` | 배경, 여백, 반전 텍스트 |
| WWF Orange | `#FF6B00` | CTA 버튼, 강조 포인트, 액센트 |
| Warm Gray | `#F5F3F0` | 섹션 배경, 카드 배경 |
| Dark Gray | `#333333` | 보조 텍스트, 서브헤딩 |
| Light Gray | `#999999` | 캡션, 비활성 요소 |

**원칙:**
- 오렌지는 CTA와 핵심 강조에만 사용한다. 남용하지 않는다
- 기본 톤은 블랙+화이트의 대비로 만든다
- 미션별 랜딩페이지에서 사진의 컬러가 주도하되, UI 컬러는 통일한다

### 6.3 타이포그래피

에디토리얼 톤에 맞는 서체 조합을 사용한다.

| 용도 | 서체 | 비고 |
|------|------|------|
| 디스플레이/헤드라인 (한글) | Noto Serif KR | 진중하고 격조 있는 세리프 |
| 디스플레이/헤드라인 (영문) | Playfair Display | 에디토리얼 분위기의 클래식 세리프 |
| 본문 (한글) | Pretendard | 가독성 높은 현대적 산세리프 |
| 본문 (영문) | Pretendard | 한글과 통일감 |
| 캡션/UI | Pretendard | 작은 사이즈에서의 명료함 |

**타이포 스케일:**

| 요소 | 사이즈 | 굵기 | 행간 |
|------|--------|------|------|
| Hero 타이틀 | 56–72px | Bold | 1.1 |
| 섹션 헤딩 | 36–48px | Bold | 1.2 |
| 서브헤딩 | 24–28px | Medium | 1.3 |
| 본문 | 18–20px | Regular | 1.7 |
| 캡션 | 14px | Regular | 1.5 |
| CTA 버튼 | 16–18px | Bold | — |

**원칙:**
- 헤드라인은 크고 담대하게. 매거진 표지처럼 타이포 자체가 비주얼 요소가 된다
- 본문은 넉넉한 행간(1.7)으로 읽기 편안하게
- 한 화면에 3단계 이상의 타이포 위계를 두지 않는다

### 6.4 레이아웃 원칙

**전체 구조:**
- 최대 콘텐츠 너비: 1200px (본문 텍스트는 720px 이내)
- 사진은 전폭(full-bleed)으로 사용하여 몰입감을 높인다
- 텍스트와 이미지의 비율은 약 3:7 — 사진이 주도하고, 텍스트는 절제한다

**미션 랜딩페이지 레이아웃 패턴:**
- 히어로: 전면 사진 + 오버레이 타이틀 (화면 100% 높이)
- 내러티브 섹션: 사진-텍스트 교차 배치, 스크롤 기반
- 인용/통계: 대형 숫자나 인용구를 독립 섹션으로 강조
- CTA: 하단 고정이 아닌 내러티브 흐름의 자연스러운 끝에 배치

**카드 디자인 (메인 랜딩):**
- 사진 비율: 3:4 세로형
- 카드 위에 미션명 오버레이
- 호버 시 사진 줌 + 한 줄 설명 페이드인
- 6개 카드는 2×3 또는 3×2 그리드

### 6.5 사진 방향성

- 자연 다큐멘터리급 사진 품질을 지향한다
- 동물 클로즈업보다는 **서식지와 환경 전체를 담는 와이드 샷** 위주
- 사람이 등장할 경우 WWF 활동가나 지역사회 모습 (스톡 사진 느낌 배제)
- 프로토타입에서는 Unsplash/Pexels의 고품질 자연 사진으로 대체

**이미지 해상도 기준:**

| 용도 | 비율 | 최소 해상도 | next/image 설정 |
|------|------|------------|----------------|
| 메인 히어로 | 16:9 | 1920×1080 | priority, placeholder="blur" |
| 미션 히어로 (풀스크린) | 16:9 | 1920×1080 | priority, placeholder="blur" |
| 미션 카드 (메인 랜딩) | 3:4 | 600×800 | lazy, sizes="(max-width: 768px) 100vw, 33vw" |

### 6.6 인터랙션 & 모션

**스크롤 내러티브:**
- 섹션 진입 시 텍스트 페이드업 (staggered animation)
- 사진은 패럴랙스 또는 스케일 트랜지션
- 통계 숫자는 카운트업 애니메이션

**기부 스텝 모달:**
- 모달 오픈: 하단에서 슬라이드업
- 스텝 전환: 좌우 슬라이드 (자연스러운 진행감)
- 스텝 인디케이터: 상단 프로그레스 바 (얇고 미니멀하게)

**버튼 & 호버:**
- CTA 버튼: 호버 시 배경색 진해짐 + 미세한 스케일업
- 카드: 호버 시 사진 줌(1.05) + 오버레이 텍스트 등장
- 모든 전환: ease-out, 300–500ms

### 6.7 반응형 기준

| 구간 | 너비 | 비고 |
|------|------|------|
| Desktop | 1200px+ | 기본 레이아웃 |
| Tablet | 768–1199px | 2열 그리드 → 카드 재배치 |
| Mobile | ~767px | 1열, 터치 최적화 |

## 7. UX 핵심 원칙

### 7.1 상품화 방지

이 플랫폼의 가장 중요한 UX 원칙이다.

- 기부 카테고리에 **가격을 직접 붙이지 않는다** — "해양 보전 50,000원" (X)
- 금액 설정은 카테고리 선택과 분리된 별도 단계에서 이루어진다
- 카테고리 선택 화면에서 금액이 노출되지 않는다
- "장바구니"를 연상시키는 UI 패턴을 사용하지 않는다 (수량 조절, 합계 계산 등)
- 후원품/리워드 안내는 기부 플로우 내에서 노출하지 않는다

### 7.2 감사와 긍정

- 모든 선택지에 긍정 메시지를 부여한다
- "아니오" 선택이 "덜 좋은 선택"으로 느껴지면 안 된다
- 기부 완료 후 메시지는 금액 크기와 무관하게 동일한 감사의 톤을 유지한다
- 비교나 랭킹을 만들지 않는다 ("상위 10% 기부자" 등 배제)

### 7.3 카피 톤

**전체 톤:** 차분하고 진중하다. 과장하지 않고, 감정에 호소하되 사실에 기반한다.

**피해야 할 표현:**
- "지금 당장" "서둘러" 등 긴급성을 과도하게 강조하는 표현
- "단 1만원이면" 등 금액을 축소하는 표현
- "당신이 아니면 안 됩니다" 등 죄책감을 유발하는 표현

**지향하는 표현:**
- "당신의 마음이 닿는 곳"
- "지금, 자연이 보내는 신호"
- "함께 지켜가는 이야기"
- "당신의 후원이 만드는 변화"

### 9.5 GSAP 애니메이션 전략

**미션 랜딩페이지 스크롤 내러티브:**
- ScrollTrigger를 사용한 섹션 단위 애니메이션
- 텍스트: 하단에서 페이드업 (stagger 0.1s)
- 사진: 패럴랙스 스크롤 (speed 0.5) 또는 스케일 트랜지션 (1.1 → 1.0)
- 통계 숫자: 뷰포트 진입 시 카운트업 애니메이션
- 핀 섹션: 사진 고정 + 텍스트 스크롤 오버레이 (선택적)

**메인 랜딩 카드:**
- 뷰포트 진입 시 staggered 페이드업 (각 카드 0.15s 딜레이)
- 호버: CSS transform으로 처리 (GSAP 불필요)

**기부 모달:**
- 모달 자체: CSS 트랜지션으로 충분 (GSAP 미사용)
- 스텝 전환: CSS 슬라이드 트랜지션

**주의사항:**
- GSAP은 클라이언트 컴포넌트(`'use client'`)에서만 사용
- `useLayoutEffect` + `gsap.context()`로 클린업 관리
- ScrollTrigger는 `useEffect` 내에서 초기화

## 10. 접근성 (프로토타입 기본 범위)

프로토타입 단계에서도 아래 기본 접근성을 보장한다. 전수 검증(WCAG AA 전체 항목)은 v2.0 범위이다.

### 10.1 프로토타입 필수 범위

| 항목 | 기준 | 구현 방법 |
|------|------|-----------|
| 키보드 네비게이션 | 모든 인터랙티브 요소 Tab 접근 가능 | 시맨틱 HTML (`<button>`, `<a>`, `<input>`) 사용 |
| 모달 포커스 트랩 | 기부 모달 열림 시 포커스가 모달 내부에 갇힘 | shadcn/ui Dialog의 기본 포커스 트랩 활용 |
| 모달 Esc 닫기 | Esc 키로 모달 닫기 | Dialog 기본 동작 |
| alt 텍스트 | 모든 `<img>`에 의미있는 alt 제공 | `NarrativeImage.alt` 필드 활용 |
| 색 대비 | 텍스트/배경 대비 4.5:1 이상 (WCAG AA) | Black/White 기본 조합으로 충족, Orange CTA 확인 필요 |
| 폼 레이블 | 모든 입력 필드에 연결된 label | `<label htmlFor>` 또는 `aria-label` |
| 에러 메시지 연결 | 검증 실패 시 스크린리더 인식 | `aria-describedby` + `aria-invalid` |
| 애니메이션 감소 | 시스템 설정 존중 | `prefers-reduced-motion: reduce` 시 GSAP 애니메이션 비활성화, CSS 전환만 유지 |

### 10.2 구현 가이드


## 11. 로딩 상태 및 성능 목표

### 11.1 로딩 상태 UI

| 상황 | 로딩 UI | 설명 |
|------|---------|------|
| 페이지 전환 | 상단 프로그레스 바 (얇은 오렌지 라인) | Next.js `loading.tsx` 활용 |
| 히어로 이미지 로딩 | blur placeholder → 선명 이미지 | next/image `placeholder="blur"` + `blurDataURL` |
| 내러티브 이미지 로딩 | 회색 배경 → 페이드인 | lazy loading + opacity 전환 |
| PDF 생성 중 | 버튼 내 스피너 + "기증서 생성 중..." | 버튼 비활성화, 완료 시 자동 다운로드 |
| 기부 완료 처리 | 버튼 내 스피너 + "처리 중..." | "후원하기" 버튼 비활성화 |

### 11.2 성능 목표 (Core Web Vitals)

| 지표 | 목표 | 측정 조건 |
|------|------|-----------|
| LCP (Largest Contentful Paint) | < 2.5초 | 히어로 이미지 기준 |
| FID (First Input Delay) | < 100ms | 모바일 기준 |
| CLS (Cumulative Layout Shift) | < 0.1 | 이미지 로딩 시 레이아웃 밀림 방지 |
| TTI (Time to Interactive) | < 3.5초 | GSAP 초기화 포함 |

### 11.3 성능 최적화 전략

| 전략 | 적용 대상 | 방법 |
|------|-----------|------|
| 이미지 최적화 | 전체 이미지 | next/image 자동 최적화 (WebP/AVIF), srcSet 자동 생성 |
| 히어로 우선 로딩 | 메인/미션 히어로 | `priority` 속성, `fetchpriority="high"` |
| GSAP 지연 로딩 | 미션 랜딩페이지 | `next/dynamic`으로 GSAP 포함 컴포넌트 동적 import |
| 폰트 최적화 | 전체 | `next/font`로 셀프호스팅, `display: swap` |
| PDF 라이브러리 분리 | 감사 페이지 | `@react-pdf/renderer`를 동적 import (번들 분리) |

## 12. 프로토타입 제외 범위
